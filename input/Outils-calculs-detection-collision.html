Outils mathématiques pour la détection des collisions en 2D

Présentation intuitive

Ce sujet peut être relativement complexe à résoudre si on ne dispose pas des bons outils.Cette page propose quelques outils mathématiques souvent utiles pour traiter le sujet de la détection des collision en 2D.

Ce sujet relève beaucoup plus de l'algorithmique que des mathématiques, car devant être implémenté dans un langage où un autre. Il existe certaines méthodes plus ou moins documentées. Ici, vous trouverez à la fois un contenu théorique en mathématiques et des iens vers des ressources connexes.

L'objectif est de vous permettre de disposer d'une palette d'outils et de méthodes qui vous permettront de coder votre propre Mine Storm par exemple. Même si l'bjectif n'est pas de revenir dans les années 80 (1980), le but est de comprendre comment développer son propre outil de détection de collision.

Cette page est évolutive, ses sources sont dans un Répertoire GitHub.

Conclusion : pour localiser un point sur un axe, on peut l'écrire de plusieurs manières.
Première écriture : \begin{align} &M = (x_{M}) ; \end{align} Ou une écriture vectorielle : \begin{align} &\overrightarrow{OM} = x_{M}.\overrightarrow{i} ; \end{align} Ces deux écritures définissent de deux manières différentes bien qu'équivalentes le point $M$.

Norme des vecteurs

Ou comment calculer des distances. Dans le plan, on dispose de deux points, $A$ et $B$ avec comme coordonnées : \begin{align} & A= (x_{A} ; y_{A}) && \text{ coordonnées de $A$ dans le plan }\\ & B= (x_{B} ; y_{B}) && \text{ coordonnées de $B$ dans le plan } \end{align} Dans le plan, la distance entre les deux points est une application directe du théorème de pythagore. A l'aide des coordonnées des deux points, la distance entre $A$ et $B$, notée $AB$ vaut : \begin{align}\label{eqdist2d} &|AB| =\sqrt{(x_{B}-x_{A})^{2} + (y_{B}-y_{A})^{2}} \end{align}
